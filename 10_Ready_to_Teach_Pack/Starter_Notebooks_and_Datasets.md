The course uses notebooks as reproducible workspaces, not as slide decks. Each starter begins with a task contract and assertions, contains incomplete implementation cells, and ends with a fresh-run checklist. Students must independently rebuild the pattern after guided practice.

## Common Notebook Header

Every notebook starts with:

```python
from __future__ import annotations

import platform
import random
from pathlib import Path

import numpy as np

SEED = 42
random.seed(SEED)
np.random.seed(SEED)

print('Python:', platform.python_version())
print('Working directory:', Path.cwd())
```

PyTorch notebooks additionally set the seed and report device/package versions. Competition notebooks must not install packages silently during execution.

## Common Cell Sequence

1. Task and metric in the student's own words.
2. Imports, versions, paths, and seed.
3. File/schema audit.
4. Split/validation decision before model training.
5. Constant or rule baseline.
6. First complete model pipeline.
7. Official metric plus diagnostic metrics.
8. Error/slice analysis.
9. One controlled experiment per cell group.
10. Final fit/inference and submission validation.
11. Fresh-runtime run and output hash.
12. Reflection/error/AI-use record.

## Dataset Pack

### D01 — Python and Statistics Tables

Small inline tables for code tracing, descriptive statistics, confusion matrices, scaling, and distance. These remain visible because they are instructional rather than scored competition data.

### D02 — Mixed-Type Tabular Practice

Fields: `record_id`, two numerical features, two categorical features, one missingness pattern, one group, and a binary target. It supports mixed preprocessing, group validation, metrics, and threshold selection.

### D03 — Scientific Regression Practice

Fields: positive/skewed measurements, material category, experiment group, missing values, nonlinear interactions, and continuous target. It supports log/ratio features, grouped validation, linear/tree models, and residual slices.

### D04 — Synthetic Shape Images

Generated locally with circles, rectangles, lines, noise, rotation, and variable contrast. It supports OpenCV baselines, CNNs, detection-style box calculations, and leakage demonstrations through source styles.

### D05 — Text Classification Practice

Short generated messages with class vocabulary, negation, ambiguous cases, duplicates, long examples, and source fields. It supports TF-IDF, vocabulary/padding, LSTM, transformers, and source/duplicate leakage analysis.

### D06 — Audio Practice

Generated tones/chirps/noise and optionally teacher-supplied real clips. It supports sample-rate audits, waveforms, Mel features, source-aware splitting, and classification without redistributing copyrighted recordings.

### D07 — Diagram–Caption Multimodal Practice

Locally generated geometric diagrams paired with true/false captions. It supports structural image features, text features, fusion, source stress tests, and submission validation.

## Scored Mock Data Rule

Practice generators may be visible. Hidden-label scored mocks are generated by the teacher and distributed as CSV/image/audio inputs only. Label-generating code and hidden labels remain in the private teacher pack. This prevents students from reverse-engineering the target formula.

## Fresh-Run Acceptance Criteria

A notebook passes only when:

- kernel/runtime is restarted;
- all cells run in order without manual state repair;
- input files are located through relative/configured paths;
- random seeds and package versions are recorded;
- outputs and submissions are regenerated;
- assertions verify row counts, columns, ranges, shapes, and finite values;
- no secret, hidden label, or prohibited asset is committed;
- the student can orally explain a teacher-selected cell.
