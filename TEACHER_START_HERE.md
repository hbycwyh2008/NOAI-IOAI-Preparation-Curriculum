# Teacher Start Here

This curriculum is organised around the NOAI syllabus rather than around any single textbook or video course.

## Core Design Rules

- One required resource per mission; optional support is limited to one or two items.
- English resources are preferred.
- First-round lessons must include paper-based questions, concept explanations, calculations, code reading, or code completion.
- Second-round lessons must produce executable notebooks and model-evaluation evidence.
- Complete solutions and scoring keys remain in the separate private teacher-keys repository.

## Recommended Implementation Order

1. Read [Course Map](00_Course_Overview/Course_Map.md).
2. Check the [NOAI Syllabus Map](00_Course_Overview/NOAI_2026_Syllabus_Map.md).
3. Run the student diagnostic.
4. Assign one class mission at a time.
5. Use [Evidence System](04_Assessment/Evidence_System.md) and [Readiness Checklist](04_Assessment/Readiness_Checklist.md).
6. Schedule Round 1 and Round 2 mock contests.

## Separation of Repositories

- **Curriculum repository:** student-facing missions, resources, templates, starter code.
- **Teacher keys repository:** answers, reference implementations, scoring notes, hidden test cases.
